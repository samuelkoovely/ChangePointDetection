# Figure Reproduction

Each Python-rendered manuscript figure has a numbered script at the repository root. Run `fig_x.py` for manuscript Figure `x`. Figure 2 is generated directly in LaTeX and has no Python script.

The numbered files are the plotting implementations, so existing command-line options live directly on those scripts. For example, `python fig_1.py --output figures/custom.pdf` writes Figure 1 to a custom path.

## Cached Inputs

Most plotting scripts read cached pickles from `data/` and `gridsearch_results/`. If those files are already present, run only the numbered figure script. If they need to be regenerated, use the preparation commands below.

### Figures 1 and 5: Motif Entropy

```bash
python generate_motifs.py
python compute_global_entropy_motifs.py
python compute_global_entropy_motifs.py --reverse-time
python compute_entropy_motifs.py --output-base gridsearch_results/motifs_f
python split_merge_limit.py
python fig_1.py
python fig_5.py
```

`fig_1.py` uses the global motif results. `fig_5.py` combines local motif entropy with the split-merge connected-component limit.

## Suggested File Organization

The numbered manuscript figure scripts now sit at the repository root. A fuller cleanup can be done later by grouping the remaining non-numbered scripts by role:

| Directory | Contents |
| --- | --- |
| `fig_x.py` at root | Plot implementations for numbered manuscript figures. |
| Other `fig_*.py` files | Supplementary plots or shared plotting helpers not mapped to the main numbered figures. |
| `generate_*.py` | Dataset and network generation scripts. |
| `compute_*.py` | Entropy signal and other cached-feature computation scripts. |
| `gridsearch_*.py` | Training/grid-search pipelines that write `gridsearch_results/`. |
| `evaluate_*.py` | Test-set evaluation pipelines. |
| `data/` | Input datasets and generated network pickles. |
| `gridsearch_results/` | Cached training, signal, limit, and test summaries. |
| `figures/` | Rendered figure outputs. |
